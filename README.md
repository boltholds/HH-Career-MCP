# HH Career MCP

MCP-сервер для безопасной работы с карьерными данными HH.ru: поиск и анализ вакансий, подготовка адаптированных вариантов резюме, чтение откликов и чатов через официальный API.

Разработка ведётся API-first. Playwright не входит в критический контур и может использоваться позднее только как управляемый пользователем UI-helper.

## Текущий вертикальный срез

Первая версия предоставляет только read-only инструменты:

- `hh_connection_status` — проверка доступности API и наличия OAuth-токена;
- `hh_get_current_user` — чтение текущего авторизованного пользователя;
- `hh_search_vacancies` — поиск вакансий с типизированными фильтрами;
- `hh_get_vacancy` — получение полной карточки вакансии.

Отправка откликов, сообщений и изменение резюме намеренно отсутствуют. Write-инструменты будут добавляться отдельно с обязательным подтверждением пользователя.

## Быстрый запуск

Требуется Python 3.12+.

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
python -m pip install -e ".[dev]"
cp .env.example .env
```

Заполните в `.env`:

```dotenv
HH_ACCESS_TOKEN=
HH_USER_AGENT=HH-Career-MCP/0.1 (your-email@example.com)
HH_API_BASE_URL=https://api.hh.ru
HH_REQUEST_TIMEOUT_SECONDS=20
HH_MCP_TRANSPORT=stdio
```

`HH_USER_AGENT` должен содержать название приложения и рабочий контакт разработчика. Публичный поиск работает без OAuth; `/me`, резюме, отклики и чаты требуют пользовательский токен.

Запуск MCP-сервера:

```bash
hh-career-mcp
```

Проверка разработки:

```bash
ruff check .
pytest -q
```

## Архитектурные границы

```text
MCP tools
   │
   ▼
HHClient port-facing adapter
   │
   ▼
official api.hh.ru
```

MCP-слой отвечает за безопасные и понятные модели инструментов. `HHClient` инкапсулирует HTTP, заголовки, OAuth и ошибки HH.ru. Следующие сервисы анализа вакансий и адаптации резюме будут строиться поверх этого клиента, а не внутри transport-кода.

## Ближайшие этапы

1. Полный OAuth flow с безопасным хранилищем и обновлением токена.
2. Получение собственных резюме и master-resume facts store.
3. Оценка соответствия вакансии подтверждённому опыту.
4. Генерация варианта резюме и diff без публикации.
5. Чтение откликов и новых Chat API.
6. Подтверждаемые write-операции.
7. Опциональный локальный browser helper без массового скрапинга.
